## Master System Specification Prompt: Remote Configuration, Notification Engine, & Legal Boundary Matrix## 1. Dynamic Remote Configuration & Forced Update Engine
The system utilizes an API-driven Remote Configuration matrix managed directly via the Admin Dashboard. This allows real-time user interface alterations and application version enforcement without requiring an app store update.
## A. Dynamic Asset Control

* Zero-Update Logo Manipulation: The admin panel hosts and changes the application's primary brand logo image web link. The client application must fetch this link during its initialization launch phase, updating the local user interface dynamically and bypassing the need for an app store submission.
* Asset Fallback Mechanism: The client app must maintain a local default logo inside its core files. If a network timeout or connection failure occurs during launch, the app gracefully falls back to this local asset to prevent visual bugs.

## B. Version Verification & Mandatory Force-Update Flow

* API Configuration Payload: On every application boot, the client fires a request to the Admin configuration endpoint. The server returns the active operational rules, including the dynamic logo link, latest app version, minimum required version, download link, and an update status indicator.
* The Forked Launch Gate: The client application compares its current internal version against the fetched data:
* Scenario A (Up-to-Date): If the local version is greater than or equal to the minimum required version, the app updates the logo image and immediately renders the main application interface dashboard.
   * Scenario B (Outdated - Forced Update): If the local version is lower than the minimum required version, the application must immediately block user access. It intercepts the navigation routing and forces the user onto an un-skippable Forced Update Screen.
* Forced Update UI Requirements: The update interface must contain no bypass, close, or back navigation capabilities. It must present a prominent download button embedded with the admin-provided download link. Tapping this button routes the user directly to the external browser or target App Store to pull the mandatory build. If they do not download it, they cannot bypass this screen to view the application interface.

## 2. Automated & Product Notification Matrix
The platform implements a contextual behavioral engine to dispatch real-time, automated push notifications and emails based on explicit user triggers and admin actions:

* Behavioral Abandonment Triggers: If a customer launches the application, browses available catalog products, or adds items to their shopping cart but fails to complete the checkout sequence within a designated time window, the system automatically fires a high-urgency recovery notification stating that they should order before stock runs out.
* Product Marketing Broadcasts: Administrators can manually orchestrate targeted product marketing campaigns from their dashboard. These entries push rich notifications featuring custom marketing copy, item images, and direct product-page deep links straight to the user base.
* Transactional Status Updates: Automatic, system-generated operational updates dispatch instantly to the customer's device upon database order status modifications, such as order confirmation, dispatching confirmation, or order completion.

------------------------------
## 3. Admin Analytics & Click-Through Tracking
The Admin Dashboard features a centralized tracking ledger to log, monitor, and audit the real-time performance and conversion throughput of every sent notification:

* Impression Counter (Seen): Increments and registers the total volume of unique client devices where the push notification was successfully delivered and verified as rendered on the user's screen.
* Click-Through Counter (Clicked): Increments and registers the exact count of interactions where a user explicitly tapped the notification banner or accessed the embedded promotional link.
* Conversion Analytics Engine: The system automatically compiles and displays the click-through ratio calculated by dividing total clicks by total impressions. This metric provides administrators with instant clarity on message performance and audience conversion behavior.

------------------------------
## 4. Mandatory App Policies (Terms & Conditions)
To establish legal protection for the platform, the client application must display a persistent, non-negotiable set of system-wide legal policies. These are enforced globally during user registration, inside all checkout screens, and within a dedicated account settings tab:

* Strict Return Policy: The platform operates under an absolute No-Return Policy. Once an order enters processing status, items cannot be shipped back to the warehouse or reversed in the system.
* Strict Refund Policy: The platform enforces a rigorous No-Refund Policy. All financial payments, payment transactions, and in-app balance transactions are final, completely irreversible, and non-reimbursable.
* Strict Exchange Policy: The platform enforces a categorical No-Exchange Policy. Users are blocked from trading, swapping, or substituting ordered items or processed goods for alternative variations, colors, sizes, or product variants.

------------------------------
## 5. Product Description Enforcement Boundary
To eliminate customer disputes, minimize support overhead, and guarantee absolute consumer awareness before a financial commitment, the application structure forces global layout rules on all commercial items:

* Mandatory Policy Disclaimer: The exact, literal phrase "No Refund, No Exchange, No Return" is permanently fixed into the core design layout template of every single product description block. This text cannot be hidden, overwritten, or omitted by individual merchants or product creators.
* Immutable Visibility Block: This policy notice must be styled in a bold, high-contrast, prominent text block placed directly adjacent to the main order or final checkout buttons. This layout guarantees that consumers visually witness and implicitly agree to the finality of the terms before triggering any purchasing transaction.
